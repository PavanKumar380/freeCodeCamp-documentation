Introduction

HTML (HyperText Markup Language) is the basic build block of the website. It defines the meaning and structure of web content.

Hypertext refers to links that connect web pages to one another, either within a single website or between different websites.

Markup refers to the special tags, called elements, enclosed in < and >. Each of these tags has a specific meaning to the browser, as well as accessibility tools, that helps them understand the purpose of the content enclosed within the tags.

Some examples of tags include <html>, <head>, and <body>.

My hope is that this documentation helps you achieve the following:

  • Learn about the various elements of a HTML markup
  • Gain a basic understanding of how web browsers understand your HTML file
  • Gain an appreciation for the accessibility features enabled by the semantic nature of certain HTML elements
  • Feel ready to take the next step of your frontend journey: CSS
  • Have fun

HTML Boilerplate

The following are two examples of HTML boilerplates that contain tags that ensure basic functionality of a webpage. (Don't worry about what it all means. We'll get to that in the coming sections.)

Minimal: <html>
  <head>
    <title>Webpage Title</title>
  </head>
  <body>
    Webpage Content
  </body>
</html>

Recommended: <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Webpage Title</title>
  </head>
  <body>
    Webpage Content
  </body>
</html>

Linked with a css file named styles.css (you won't need this until you start learning CSS): <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="styles.css">
    <title>Webpage Title</title>
  </head>
  <body>
    Webpage Content
  </body>
</html>

Elements and Attributes

An element in HTML is a basic unit of the markup that has its own purpose as understood by the browser. An element in HTML is represeted by its tag.

You may have noticed that some elements in the boilerplate have extra information specified within the tag. For example, <html lang="en">. In this case, the extra information lang is called an attribute. It has been defined with a value of en. Giving the <html> element an attribute of lang="en" lets the browser (and accessibility tools) know that your webpage is in English.

You may also have noticed that some tags in the boilerplate come in pairs, like <html> and </html>, while other tags are individual, like <meta>. The paired tags are called container tags, while the individual tags are called empty tags, void tags, or self-closing tags.

Container tags, like the name suggests, are meant to hold content between their opening and closing tag.

Empty tags perform various functions either by themselves or based on attributes defined in them.

In the next few sections, we shall examine and understand what each element in the boilerplate is.

The DOCTYPE Element

The <!DOCTYPE html> element is most commonly the first element in any standard html file. It lets the browser know that it is dealing with a html file.

The html Element

This element contains all the markup required for your webpage within its opening tag, <html>, and its closing tag </html>. All your HTML markup should be within those tags.

It is also standard practice to define the language of your webpage in the opening tag of this element, using the lang attribute. For example, <html lang="en">.

The head Element

The <head> element contains metadata about your webpage, which helps browsers and search engines understand the content of your webpage.
